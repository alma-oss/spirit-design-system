# AI Agent Setup Guide

This guide covers how AI coding agents should be configured for this repository and what the recommended settings are for each tool.

Teams using different AI agents need a consistent starting point for configuration.
This guide provides recommended settings for each tool so contributors can adopt a safe, shared setup without configuring everything from scratch.

👉 Please keep in mind that **you are responsible for the committed code generated by your agent**,
so always thoroughly check its output and ensure that there are no security breaches.

## Overview

| File                    | Purpose                                                |
| ----------------------- | ------------------------------------------------------ |
| `.mcp.json`             | Shared MCP server definitions (Figma, GitHub, Jira, …) |
| `.claude/settings.json` | Recommended permissions for Claude Code                |
| `.cursor/cli.json`      | Recommended permissions for Cursor CLI                 |
| `.cursor/mcp.json`      | MCP server config for Cursor                           |

## MCP Servers

[Model Context Protocol (MCP)][mcp] servers extend AI agents with project-specific tools.
Provide a set defined in `.mcp.json` at the repository root.

| Server            | Purpose                                                       |
| ----------------- | ------------------------------------------------------------- |
| `chrome-devtools` | Browser automation and debugging via Chrome DevTools          |
| `context7`        | Up-to-date library documentation for LLM context              |
| `figma`           | Read and write Figma designs; Code Connect mappings \*        |
| `github`          | Read PRs, issues, and repository metadata via GitHub API \*\* |
| `jira`            | Read and create AlmaCareer Jira issues \*\*                   |

(\*) The `figma` server connects to a locally running Figma Desktop app (`http://127.0.0.1:3845/mcp`),
so Figma must be open for it to work.

(\*\*) The `github` and `jira` servers require authentication tokens.

### Recommended `.mcp.json`

The structure is the same as Claude Code and Cursor.

```json
{
  "mcpServers": {
    "chrome-devtools": {
      "type": "stdio",
      "command": "npx",
      "args": ["chrome-devtools-mcp@latest"]
    },
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    },
    "figma": {
      "type": "http",
      "url": "http://127.0.0.1:3845/mcp"
    },
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp/",
      "headers": {
        "Authorization": "Bearer ${GITHUB_TOKEN}"
      }
    },
    "jira": {
      "type": "http",
      "url": "https://jira.almacareer.tech/mcp",
      "headers": {
        "Authorization": "Token ${JIRA_TOKEN}"
      }
    }
  }
}
```

## Claude Code

### Recommended `.claude/settings.json`

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "defaultMode": "plan",
    "allow": [
      "Bash(git *)",
      "Bash(gh pr:*)",
      "Bash(gh repo:*)",
      "Bash(gh api:*)",
      "Bash(ls:*)",
      "Bash(cat:*)",
      "Bash(echo:*)",
      "Bash(pwd)",
      "Bash(which:*)",
      "Bash(find:*)",
      "Bash(grep:*)",
      "Bash(head:*)",
      "Bash(tail:*)",
      "Bash(wc:*)",
      "Bash(diff:*)",
      "Bash(stat:*)",
      "Bash(jq:*)",
      "Bash(yarn *)",
      "Bash(make *)"
    ],
    "ask": ["Bash(git branch -D:*)", "Bash(git push --force:*)", "Bash(make version)"],
    "deny": [
      "Bash(git reset --hard:*)",
      "Bash(git clean -fd:*)",
      "Bash(git filter-branch:*)",
      "Bash(git gc:*)",
      "Bash(rm -rf *)",
      "Bash(* publish:*)",
      "Read(~/.ssh/*)",
      "Read(.env*)",
      "Write(.env*)"
    ]
  },
  "attribution": {
    "commit": "",
    "pr": ""
  },
  "enabledMcpjsonServers": ["chrome-devtools", "context7", "figma", "github", "jira"]
}
```

### Key Settings Explained

**`defaultMode: plan`** — Agents start in plan mode and propose a plan before executing anything. Switch per session with `/auto` or `/code`.

**`allow`** — Safe, everyday operations that run without confirmation: `git`, `gh`, read-only shell commands, `yarn`, `make`.

**`ask`** — Potentially destructive but sometimes needed: `git push --force`, `git branch -D`. Claude will prompt before running these.

**`deny`** — Irreversible or sensitive operations that are never allowed: `git reset --hard`, `git clean -fd`, `rm -rf`, publish commands, access to `.env*` and `~/.ssh/*`.

**`attribution`** — Empty strings disable AI attribution footers in commits and PRs (see [Git Workflow Guidelines][agents-git-workflow]).

**`enabledMcpjsonServers`** — Activates the servers defined in `.mcp.json`.

### Local Overrides

For personal allow rules (e.g. tools specific to your machine), use `.claude/settings.local.json` (gitignored):

```json
{
  "permissions": {
    "allow": ["Bash(my-custom-tool:*)"]
  }
}
```

## Cursor CLI

Cursor settings mirror the Claude Code setup in a Cursor-compatible format.

### Recommended `.cursor/cli.json`

```json
{
  "permissions": {
    "allow": [
      "Shell(git)",
      "Shell(gh:pr)",
      "Shell(gh:repo)",
      "Shell(gh:api)",
      "Shell(ls)",
      "Shell(cat)",
      "Shell(echo)",
      "Shell(pwd)",
      "Shell(which)",
      "Shell(find)",
      "Shell(grep)",
      "Shell(head)",
      "Shell(tail)",
      "Shell(wc)",
      "Shell(diff)",
      "Shell(stat)",
      "Shell(jq)",
      "Shell(yarn)",
      "Shell(make)"
    ],
    "ask": ["Shell(git branch -D)", "Shell(git push --force)", "Shell(make version)"],
    "deny": [
      "Shell(git reset --hard)",
      "Shell(git clean -fd)",
      "Shell(git:filter-branch)",
      "Shell(git:gc)",
      "Shell(rm -rf *)",
      "Shell(* publish:*)",
      "Read(~/.ssh/*)",
      "Read(.env*)",
      "Write(.env*)"
    ]
  }
}
```

### Recommended `.cursor/mcp.json`

`.cursor/mcp.json` is same as `.mcp.json` for Claude Code, so please refer to the [MCP Servers](#mcp-servers) section above.

### Local Overrides

For additional personal MCP servers, add them to `~/.cursor/mcp.json`. For permission overrides, use Cursor's settings UI (`Cursor Settings → Rules`).
