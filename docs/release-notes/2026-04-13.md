# 2026-04-13 Spirit Design System Release Notes

📢 🚀 🎉 Ahoy, we have a new release with features and improvements. Here we go.

## General Changes

We've added a wave of new building blocks this release. The headline additions are `FileUpload` and `UNSTABLE_File` for complete file handling, and `Picker` in React / `UNSTABLE_Picker` in Web for selection-based interfaces.

In React, you can now integrate client-side routing via the new `RouterProvider`. Wrap your app once with your router's `navigate` function and all Spirit link-like components (`Link`, `ButtonLink`, …) will automatically use it for internal navigation instead of triggering a full page load.

```tsx
// e.g. Next.js App Router — app/layout.tsx
'use client';
import { useRouter } from 'next/navigation';
import { RouterProvider } from '@alma-oss/spirit-web-react';

export default function Layout({ children }: { children: React.ReactNode }) {
  const router = useRouter();

  return <RouterProvider navigate={router.push}>{children}</RouterProvider>;
}
```

We've also added `FieldDetails` to enrich form fields with additional metadata, and `Textarea` now comes with a built-in character counter.

Two feature-flagged expansions also land in this release, letting you opt in early without breaking existing behavior: an extended size scale for `ControlButton` and a refreshed `Tag` appearance.

## What's Next 🔮

- **Interactive Tag**
  Allowing the `Tag` component to be interactive (button or link) with dynamic hover and active colors, and aligned sizing across `Tag`, `Input`, and `Button` for the upcoming v5.
- **Custom Select / Multiselect**
  A new custom select/multiselect component for advanced selection and filtering use cases, currently in prototyping.
- **Input Addons**
  Slot-based input addons starting with a refactored `PasswordToggle` as the first `InputAddon`, enabling flexible icon and action placement inside form fields.
- **Outline Form Fields**
  An "outline" variant for form field components, giving teams another visual style option for forms.
- **Picker Enhancements**
  Extended `Picker` examples covering groups, dividers, confirm buttons, and radio options, plus updated design aligned with the latest graphic specs.
- **Non-modal Dialog (React)**
  Bringing the non-modal `Dialog` and `Drawer` pattern (already in Web) to React consumers.
