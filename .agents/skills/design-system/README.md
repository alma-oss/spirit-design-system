# Design System

Foundational knowledge of the Spirit Design System, used by every code-review perspective.

## Purpose

Tells a reviewer what "correct for Spirit" means: the dual `web` / `web-react` structure and the
parity expectation between them, design-token usage and structure, theming rules (no
light/dark/inverted naming), the prop-vs-token customization decision, naming and file conventions,
and deprecation expectations. It is knowledge, not a process.

## Usage

Always loaded (via `cat`) by every perspective reviewer on top of the code-review methodology. Enrichment of
token/theme specifics from published Spirit knowledge (skills.sh) is planned.

## Related Skills

- the code-review methodology — shared methodology this layers on.
- `spirit:scss`, `spirit:react`, `spirit:typescript` — technology lenses that build on these rules.
- `spirit:component-deprecation` — the full deprecation workflow.
